Diagnosis
- Relevant error lines: [line 66](#L66) shows the file that failed, and [line 67](#L67) is the actual ESLint error: Do not use an `<a>` element to navigate to `/`. Use `<Link />` from next (this caused the build to exit non-zero). The build stop is shown at [line 63](#L63) and the failing build command at [line 84](#L84)-[line 88](#L88).
- Error type and cause: an ESLint rule enforced by Next.js flagged a navigation issue in ./src/app/connect/progress/page.tsx: you are using a plain <a> to navigate internally (to "/") instead of next/link's <Link /> component. Next.js runs ESLint during production builds and ESLint errors cause next build to fail by returning a non-zero exit code.

Solution
1) Preferred fix â€” replace the plain anchor with next/link
- Open the file in the repo: https://github.com/QuarkLCloudKids/quarklcloud/blob/main/src/app/connect/progress/page.tsx
- Find the anchor used for client-side navigation (the error points to [line 67](#L67), which is the "11:11" position in that file).
- Replace it with Next's Link component. Example change:

```tsx
// before
<a href="/">Go home</a>

// after
import Link from 'next/link';

<Link href="/">Go home</Link>
```

- Commit and push the change, then trigger a new Netlify build.

2) Temporary/workaround â€” disable ESLint run during the build (not recommended long-term)
- If you cannot fix all lint errors immediately, you can tell Next to ignore ESLint during builds by adding/updating next.config.js at the project root:

```js
// next.config.js
/** @type {import('next').NextConfig} */
const nextConfig = {
  eslint: {
    ignoreDuringBuilds: true
  }
}

module.exports = nextConfig;
```

- Commit and push. This will allow the build to succeed while you fix the lint issues locally. Note: this only skips linting during build and does not fix the underlying code issue.

Why this fixes it
- Next.js enforces certain accessibility/navigation rules via ESLint in production builds. Replacing <a href="/"> with <Link href="/"> satisfies the rule and prevents the ESLint error that made next build return a non-zero exit code (see [line 67](#L67)).
